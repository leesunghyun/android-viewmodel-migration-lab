# ROADMAP

## Mission

Help maintainers and practitioners migrate legacy Android shared-state architecture to Kotlin + Compose + ViewModel + StateFlow in a safe, incremental way.

## Versioned Roadmap

### v0.2 (Current)
- Finalize and validate submission readiness artifacts.
- Add maintainer-facing operational documents.
- Capture release evidence and build reliability signals.
- Improve migration task discoverability via issue templates and labels.

### v0.3
- Add one additional migration scenario that demonstrates backward-compatible state migration.
- Publish a second guided PR pack with edge-case coverage and anti-pattern fixes.
- Add a stronger review checklist and mentoring flow for first-time contributors.

### v1.0
- Declare migration lab stable for open-source onboarding.
- Expand community proof channels and external reference links.
- Keep the same project scope; avoid framework-complexity creep.

## Operating commitments

- Keep PR scope to one clear objective.
- Keep core behavior changes with tests and docs in the same PR.
- Keep maintainability evidence updated (README, changelog, release notes, release guide).
- Keep `README` consistent with active implementation and release state.
