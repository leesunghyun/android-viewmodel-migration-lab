# Security Policy

This project is an educational Android migration sample.

If you find a security issue, please open a GitHub issue with:

- clear description
- reproduction steps
- affected file/PR/release scope
- Android version and environment

Do not include secrets, API keys, private credentials, or personal data.

## Reporting workflow

1. Label as `security` in issue title or body.
2. Include severity as one of: `low`, `medium`, `high`.
3. Provide workaround if available.
4. Maintainers review and reply with triage status.

## Response expectations

- `high`: acknowledged within 3 business days.
- `medium`: acknowledged within 5 business days.
- `low`: acknowledged within 7 business days.

For immediate credential leaks, rotate credentials immediately and file a separate incident note.
